# Data Privacy & Confidentiality Reflection

## Key Takeaways from Focus Bear’s Privacy Policy

- Focus Bear collects user data to improve services and app functionality.
- Personal information is handled securely and is not sold to third parties.
- Data is shared only with trusted service providers when necessary.
- Security measures are implemented to protect user information from unauthorized access.
- Users have rights regarding their data, including access, correction, and deletion.
- Transparency and user trust are central to Focus Bear’s data practices.

---

## Confidential Data at Focus Bear

Confidential data includes:

- Information from user accounts (names, email addresses, login credentials)
- Usage and behavioural data
- Internal documentation and product roadmaps
- Business strategies and financial information
- Employee information
- API keys, passwords, and access tokens

Any non-public company or user information should be treated as confidential.

---

## Best Practices for Handling Confidential Data

- Use strong, unique passwords and enable multi-factor authentication (MFA).
- Never share login credentials.
- Store sensitive data in secure, approved platforms.
- Avoid downloading confidential data onto unsecured personal devices.
- Lock devices when away from your workspace.
- Share information only on a need-to-know basis.
- Double-check recipients before sending sensitive emails.

---

## Responding to a Suspected Data Breach

If I suspect a data breach or accidental disclosure:

1. Immediately limit further exposure (e.g., revoke access if possible).
2. Notify a manager or relevant team member immediately.
3. Clearly document what happened.
4. Do not attempt to hide the issue.
5. Follow company incident response procedures.

Prompt reporting reduces risk and protects user trust.

---

## How I Will Handle Data Securely in Daily Tasks

To ensure secure data handling, I will:

- Use only approved tools and systems.
- Avoid storing sensitive data locally unless necessary.
- Keep software updated.
- Log out of shared systems after use.
- Be cautious of phishing attempts and suspicious links.

---

## Safe Storage, Sharing, and Disposal

- Store data in secure cloud platforms with restricted access.
- Share files only via approved channels.
- Avoid unsecured public Wi-Fi when accessing sensitive systems.
- Permanently delete data when no longer required.

---

## Common Mistakes That Lead to Privacy Issues

Common mistakes include:

- Sending emails to the wrong recipient
- Reusing weak passwords
- Leaving devices unlocked
- Oversharing internal information
- Failing to report incidents quickly

These can be avoided through awareness and consistent security habits.

---

## Habit I Will Adopt

I will consistently use multi-factor authentication (MFA) and verify recipients before sending any confidential information.

---

## Key Learning

Data privacy is not only a technical responsibility but a behavioural one. Small, consistent security practices help protect user trust and company integrity.
