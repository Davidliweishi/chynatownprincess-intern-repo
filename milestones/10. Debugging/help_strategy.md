*Spend 30 minutes talking with Chat GPT to understand different perspectives on using AI in coding. When is it helpful and when is it not?*

What ChatGPT is good for:

1. Repeatative Boilerplate templates: great for generating standard code templates, which saves time on writing routine tasks. 
2. Learning and explaining: Like all AI tools, it is excellent for explaining unfamiliar concepts, in fac it can customise notes that suit your learning abilities and understanding. 
3. Refactoring: Can help give suggestions for refactoring or re-structirng your code, great for 'what-if' situaitons where you want to maximise efficieny through code suggestions. 

What ChatGPT isn't good for:

1. Complex and large code bases: it has a tendency to 'hallucinate' or misunderstand larger code bases. Therefore, not great for complicated projects that require more meticulous review. 
2. Security: It tends to overlook basic security measures in code, which can lead to dangerous and insecure code if used in main, actual codebase. 
3. Unfamiliar code library: if ChatGPT has never encountered any libraries or methods you are using, it will start making up its' own logic and hallucinate.

# Write a reflection in help_strategy.md:

**When do you prefer using AI vs. searching Google?*

When to use AI:

1. To write repetative or boilerplate code templates - which saves time
2. Explaining unfamiliar code and concepts - can tailor to your understanding and give specific contexts related to your code. 
3. Ask for specific refactoring and suggestions

When to search Google:

1. When you are dealing with complex, large codebases and require particular knowledge. 
2. When your code base has highly sensitive code and information, avoid putting in ChatGPT as it can leak to the wider net. 
3. When your code base has new libraries or codebases that are new or niche. 


**How do you decide when to ask a colleague instead?* 

I ask a colleague when I've exhausted all of my options (e.g When AI and Google search yields no results). Perhaps there is a gap in my knowledge base that I wasn't aware of. Therefore, it is better to ask someone directly. I would also ask a colleague if the codebase contains highly sensistive information that I cannot reveal in external circumstances. 


**What challenges do developers face when troubleshooting alone?* 

They may face burning out, or being limited to their own assumptions. It is useful during these times to allow an external source (whether its' AI, Google Search or asking a workmate) to verify and check your work. Developers would isolate themselves if they do not seek external help, and limit their own progress as a result.
