## Which AI tools did you try?
I tried all of them: Chatgpt, Claude AI and Github copilot

The difference I found between them:

1) Chatgpt - I found Chatgpt alot faster and more in depth for code debuging, espeiclally for smaller tasks. It is quite good at scrapping the web for answers, if you are looking for quick solutions.
2) Claude AI - I found Claude AI is great fot bigger more complicated projects a debuggings. It is quite useful if you are looking for more in depth explainations of concepts. Therefore, this makes it excellent for learning.
3) Github Copilot - I found the in-text editor quite handy, when I'm trying to streamline my coding tasks (I don't have to swithc between tabs). It's excellent for boiler-plate code. 

## What worked well? What didn’t?

1) Chatgpt - Even though Chatgpt seems to have more access to a wider net of web solutions and knowledge, I have found at times that it can be prone to the occasional inaccurancy. If you give it more complex tasks, its answers will begin to drop in quality. This would make it slightly unreliable when it comes to coding more complicated and bigger projects. the interface and layout is a little confusing sometimes, and I find myself double checking on the words I read.
2) Claude AI - Claude AI is overall better at providing more nuanced and specialised solutions to complex coding tasks. Therefore, it has stronger writing capabilities. The interface is set up better and wider, making it easier for me to read displayed content. The downside is, if I wanted to access more top-tier models with more capabilities, I would have to pay for it. At this stage it does not seem worth it. Vague instructions can lead to the wrong answers, so it was important for me to give detailed input.
3) Github Copilot - Github copilot works well, if your prefered terminal is VSC. Whether this is a downside or not is depenedent on the user's preference. It does increase my productivity in terms of typing code. However, a downside I found is that I sometimes become too reliant on it. If I use it too often, I may find myself not wanting to research my own coding solutions and just letting the AI take over. This wouldn't be good for building fundamental skills. I will continue to use it, but I will only allow it to help write code that i have understood first.

## When do you think AI is most useful for coding?

AI is good under a few circumstances:

1) Streamlining routine and repetative tasks: Tasks like generating boilerplate templates can be alot quicker - eliminating the need to do so manually.
2) Learning: Rather than look for resources yourself, you can simply ask it to scrap the web to give you tailored and personalised breakdown of concepts you like to learn/understand.
3) Small debugs: Can help you save time on small, hard-to miss- bugs, rather than manually spend too much time looking through everything.
4) Review code: Ask it for suggestions to optimise and refine your coding.
5) Prototyping code: Ask it to help generate fake, but working code that can help demostrate similar capabilities to your code. Purpose is to build rapid proof-of-concepts.
