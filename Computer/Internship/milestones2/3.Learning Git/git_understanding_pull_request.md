## Why are PRs important in a team workflow?

PRs are important because it will only allow code to merge with a main codebase/project, after it has been reviewed and debugged. This prevents any mistakes or troubleshooting from work you aren't familar with (e.g your team mates code). 

What makes a well-structured PR?

A well-structured PR consists of a small, single-task focused and easy to read code base. It mostly consists of less than 200 line code block. It must also consist of a clear title, a detailed and easy to follow description detailing what changes were made. Proper PRs should be able to pass automated reviews and are also self-reviewed as well.
